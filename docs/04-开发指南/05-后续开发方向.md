## 2. 短期开发计划（1-2个月）

### 2.1 训练记录系统完善

**目标**：完善训练记录功能，支持详细的训练日志

**功能点**：
- [ ] 训练中实时记录（组数、次数、重量、RPE）
- [ ] 训练计时器（组间休息倒计时）
- [ ] 训练完成总结页面
- [ ] 历史训练记录查看
- [ ] 训练数据导出（CSV/PDF）

**技术实现**：
```
训练记录Store
├─ 当前训练状态
├─ 实时数据同步
├─ 离线缓存支持
└─ 后台API对接
```

**目标**：提供更丰富的训练数据图表

**功能点**：
- [ ] 周/月训练量趋势图
- [ ] 各肌群训练分布饼图
- [ ] 力量进步曲线
- [ ] 训练频率热力图
- [ ] 目标完成进度条

**技术选型**：
- 图表库：ECharts 或 Chart.js
- 响应式设计：适配移动端

### 2.3 AI助手体验优化

**目标**：提升AI对话体验和响应质量

**功能点**：
- [ ] 对话历史搜索

- [ ] 训练计划预览优化
- [ ] 工具调用进度展示
- [ ] 错误重试机制

## 4. 长期开发计划（6-12个月）

### 4.1 用户自建计划功能

**目标**：让用户从动作库和食物库自主创建个性化训练和饮食计划

**功能点**：
- [ ] 训练计划创建器
  - 从动作库选择动作
  - 自定义组数、次数、重量
  - 设置训练频率和周期
  - 保存和管理自建计划
- [ ] 饮食计划创建器
  - 从食物库选择食物
  - 自定义份量和餐次
  - 计算总热量和营养素
  - 保存和管理自建饮食计划
- [ ] 计划模板库
  - 提供常见训练模板（增肌、减脂、力量等）
  - 用户可基于模板修改
  - 支持导入导出计划
- [ ] 计划分享功能
  - 分享自建计划给其他用户
  - 浏览和收藏他人的计划
  - 计划评分和评论

**技术实现**：
```
计划创建器
├─ 动作/食物选择器（多选、搜索、筛选）
├─ 参数配置面板（组数、次数、重量、份量）
├─ 计划预览和编辑
├─ 数据验证（营养平衡、训练量合理性）
└─ 后台API对接（保存、分享、导入导出）
```

**用户价值**：
- 🎯 完全自主控制训练和饮食计划
- 💪 根据个人喜好和设备条件定制
- 🤝 分享和学习他人的成功经验
- 📊 灵活调整，适应不同阶段需求

### 4.2 专业知识库功能

**目标**：聚合运动学和营养学专业知识，提供持续更新的科学依据

**功能点**：
- [ ] 知识库浏览
  - 运动学知识（肌肉解剖、动作力学、训练原理）
  - 营养学知识（营养素功能、饮食策略、补剂科学）
  - 按分类和标签组织
  - 支持关键词搜索
- [ ] 论文库
  - 收录健身相关学术论文
  - 论文摘要和关键发现
  - 引用来源和可信度标注
  - 定期更新最新研究
- [ ] 知识卡片
  - 简洁的知识点卡片
  - 图文并茂，易于理解
  - 支持收藏和分享
- [ ] AI知识问答
  - 基于知识库的专业问答
  - 引用具体知识条目和论文
  - 提供可验证的科学依据

**技术实现**：
```
知识库系统
├─ 知识图谱（Neo4j）
│   ├─ 知识节点（概念、原理、方法）
│   ├─ 论文节点（标题、作者、摘要、结论）
│   └─ 关系（支持、反驳、扩展、应用）
├─ 向量检索（Qdrant）
│   ├─ 知识向量化
│   └─ 语义搜索
├─ 内容管理
│   ├─ 知识编辑器
│   ├─ 论文导入工具
│   └─ 版本控制
└─ 前端展示
    ├─ 知识浏览器
    ├─ 论文阅读器
    └─ AI问答界面
```

**数据来源**：
- PubMed、Google Scholar等学术数据库
- 权威健身和营养学教材
- 专业健身教练和营养师的经验总结
- 用户反馈和实践案例

**用户价值**：
- 📚 学习科学的健身和营养知识
- 🔍 验证AI建议的科学依据
- 💡 了解最新的研究进展
- 🎓 提升健身专业素养

### 4.3 Neo4j知识图谱开放功能

**目标**：将Neo4j知识图谱可视化，让用户交互式探索健身知识的关系网络

**功能点**：
- [ ] 知识图谱可视化
  - 节点和关系的图形化展示
  - 支持缩放、拖拽、筛选
  - 不同类型节点的颜色和图标区分
  - 关系强度的可视化（线条粗细）
- [ ] 交互式探索
  - 点击节点查看详细信息
  - 展开/折叠相关节点
  - 路径查找（如：从某个动作到目标肌肉的训练路径）
  - 关系筛选（只显示特定类型的关系）
- [ ] 知识发现
  - 推荐相关知识节点
  - 发现隐藏的知识关联
  - 生成知识路径（如：从新手到高级的学习路径）
- [ ] 用户贡献
  - 用户可以建议新的节点和关系
  - 社区审核机制
  - 贡献者积分和排行

**技术实现**：
```
知识图谱可视化
├─ 前端可视化库
│   ├─ D3.js 或 Cytoscape.js
│   ├─ 力导向布局算法
│   └─ 交互控制（缩放、拖拽、筛选）
├─ 后端API
│   ├─ Neo4j查询接口
│   ├─ 图数据转换（Neo4j → JSON）
│   ├─ 路径查找算法
│   └─ 推荐算法
├─ 性能优化
│   ├─ 分层加载（只加载可见节点）
│   ├─ 数据缓存
│   └─ 增量渲染
└─ 用户贡献系统
    ├─ 贡献提交表单
    ├─ 审核工作流
    └─ 积分和排行榜
```

**当前知识图谱数据**：
- **4,246个节点**：动作、肌肉、器械、食物、营养素等
- **61,507条关系**：训练关系、营养关系、解剖关系等
- **高质量数据**：34字段完整的动作数据，35字段完整的食物数据

**用户价值**：
- 🔍 直观理解健身知识的关联
- 💡 发现新的训练和营养策略
- 🎓 系统化学习健身知识
- 🤝 参与知识库建设，贡献社区

**示例场景**：
1. **动作替换探索**：点击"杠铃深蹲"节点，查看相关的替代动作（腿举、哈克深蹲等）
2. **肌肉训练路径**：从"股四头肌"节点出发，查看所有训练该肌肉的动作
3. **营养素关系**：从"蛋白质"节点出发，查看富含蛋白质的食物和其功能
4. **训练计划生成**：基于图谱路径，生成科学的训练计划
