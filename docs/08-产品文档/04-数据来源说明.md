## 动作数据来源

### 数据规模

- **动作总数**: 1,790个
- **数据完整度**: 34个字段完整
- **肌肉分类**: 40个肌肉群
- **器械类型**: 涵盖杠铃、哑铃、固定器械、自重等多种类型

### 主要来源

#### 1. MuscleWiki（参考数据源）

**来源**: https://musclewiki.com

**当前使用方式**：
- 参考 MuscleWiki 公开数据进行开发和测试
- 数据已存储在本地数据库
- 用户访问时从本地数据库读取（不产生实时调用）

**数据内容**:
- 1,790 个动作数据
- 动作名称、描述、步骤
- 目标肌群分类
- 器械类型
- 肌肉图（body_map）

**来源标注**:
- 在所有动作详情页底部标注："动作数据参考：MuscleWiki (musclewiki.com)"
- 提供链接回 musclewiki.com
- 在关于页面说明数据来源

**合规计划**:

**阶段1：开发测试（当前）**
- 使用参考数据进行开发
- 添加明确的来源标注
- 声明为测试版本
- 会员收费说明为"服务器成本分摊"

**阶段2：小规模上线（月活 < 100）**
- 继续使用本地数据
- 强化来源标注
- 准备迁移到官方 API

**阶段3：正式运营（月活 > 100 或月收入 > ¥1000）**
- 订阅 MuscleWiki 官方 API
- 使用官方 API 更新数据
- 完全合规运营

**MuscleWiki 官方 API**:
- 官网：https://api.musclewiki.com
- 通过 RapidAPI 提供
- FREE 计划：3,000 次调用/月（足够一次性获取所有数据）
- PRO 计划：$29/月，20,000 次调用/月

**为什么现在不用官方 API**:
1. **成本考虑**：早期阶段没有收入，节省开发成本
2. **调用量充足**：FREE 计划 3,000 次/月足够获取所有 1,790 个动作
3. **数据缓存**：数据存储在本地，用户访问不产生 API 调用
4. **合规时机**：有稳定收入后再付费更合理

**风险控制**:
1. ✅ 添加明确的来源标注
2. ✅ 声明为测试版本
3. ✅ 准备官方 API 迁移方案
4. ✅ 设定触发条件（月活 > 100 或月收入 > ¥1000）

#### 2. 公共领域健身知识

**来源**:
- 运动生理学教材和学术文献
- 国际健身组织（如NSCA、ACSM）的公开资料
- 健身教练培训教材
- 运动解剖学标准文献

**数据类型**:
- 动作的生物力学原理
- 肌肉激活模式
- 安全注意事项
- 动作变式和替代方案

**合规说明**:
- 这些知识属于公共领域，任何人都可以学习和使用
- 我们基于科学文献进行了系统化整理
- 所有内容均经过专业审核，确保准确性

#### 3. 自主开发和整理

**原创内容**:
- 所有动作描述文本（1,790个动作的完整描述）
- 中文翻译和本地化
- 动作分类和标签系统
- 禁忌症和风险评估体系
- 动作难度评级算法
- 器械别名映射系统
- 训练参数建议（组数、次数、休息时间）

**知识图谱构建**:
- 动作与肌肉的关系网络（4,246个节点，61,507条关系）
- 动作与器械的关联
- 动作模式分类
- 训练目标映射
- 体态矫正关系（CORRECTS和AGGRAVATES关系）

**数据增强**:
- 基于运动生理学的科学验证
- 安全性评估和注意事项
- 进阶和退阶动作推荐
- 动作替代方案

## 食物数据来源

### 数据规模

- **食物总数**: 1,880个
- **营养成分**: 包含热量、蛋白质、碳水化合物、脂肪等核心营养素
- **食物分类**: 涵盖主食、肉类、蔬菜、水果、零食等多个类别

### 主要来源

#### 1. 中国食物成分表

**来源**: 《中国食物成分表》（标准版）

**出版单位**: 北京大学医学出版社

**数据类型**:
- 食物名称
- 能量（千卡）
- 蛋白质（克）
- 脂肪（克）
- 碳水化合物（克）
- 膳食纤维（克）
- 维生素和矿物质

**合规说明**:
- 《中国食物成分表》是国家权威的营养数据参考
- 数据来源于科学测定和研究
- 我们使用的是公开发布的标准数据
- 所有数据均标注了来源和参考依据

#### 2. USDA食物数据库

**来源**: USDA FoodData Central

**网站**: https://fdc.nal.usda.gov/

**数据类型**:
- 国际通用食物的营养成分
- 品牌食品的营养信息
- 食物的详细营养素分析

**合规说明**:
- USDA食物数据库是美国农业部维护的公开数据库
- 数据可免费使用，符合公共领域使用规范
- 我们对数据进行了本地化处理和单位转换

#### 3. 自主整理和补充

**数据增强**:
- 中国特色食物的营养数据补充
- 常见品牌食品的营养信息
- 食物别名和地方名称映射
- 食物分类和标签系统

**数据验证**:
- 交叉验证多个数据源
- 确保营养成分的准确性
- 定期更新和校正数据

## 合规依据

### API 使用合规

**MuscleWiki API 使用条款**：
1. **官方 API 服务**：MuscleWiki 通过 RapidAPI 平台提供官方 API
2. **商业使用许可**：付费订阅计划明确允许商业使用
3. **数据使用权**：通过 API 获取的数据可用于应用开发
4. **使用限制**：需遵守订阅计划的调用量限制

**RapidAPI 服务条款**：
- 遵守 RapidAPI 平台的服务条款
- 按订阅计划支付费用
- 不得滥用 API 或超出调用限制
- 保护 API 密钥安全

### 法律依据（规划）

#### 1. 公共领域知识

**适用范围**: 健身动作、运动原理、营养知识

**法律依据**:
- 《中华人民共和国著作权法》第五条：本法不适用于"通用数表、通用表格和公式"
- 健身动作属于人体运动的客观描述，不受版权保护
- 营养成分数据属于科学事实，不受版权保护

**说明**:
- 健身动作的名称、步骤、肌肉激活等信息属于公共领域知识
- 任何人都可以学习、使用和传播这些知识
- 我们对这些知识进行了系统化整理和结构化处理

#### 2. 合理使用原则

**适用范围**: 参考公开资料进行二次创作

**法律依据**:
- 《中华人民共和国著作权法》第二十四条：合理使用
- 为教学、科研、个人学习等目的使用他人作品

**说明**:
- 我们参考了公开的健身资料和营养数据
- 所有内容均经过重新编写和整理
- 不存在直接复制或抄袭行为

#### 3. 数据库权利

**适用范围**: 数据库的独创性编排

**法律依据**:
- 《中华人民共和国著作权法》第十四条：汇编作品
- 对数据的选择和编排具有独创性的，享有著作权

**说明**:
- 我们对健身动作和食物数据进行了独创性的分类和编排
- 建立了独特的知识图谱结构
- 开发了专有的数据关联和检索系统

### 数据使用规范

#### 1. 数据准确性保证

- 所有数据均经过专业审核
- 定期更新和维护数据质量
- 对不准确的数据及时修正

#### 2. 用户知情权

- 在帮助中心明确说明数据来源
- 用户可以查询具体数据的来源
- 接受用户对数据准确性的反馈

#### 3. 数据安全和隐私

- 用户的训练数据和个人信息严格保密
- 不会将用户数据用于商业目的
- 遵守《个人信息保护法》的相关规定

**更新频率**: 每季度一次

## 数据使用声明

### 用户权利

1. **知情权**: 用户有权了解数据的来源和准确性
2. **查询权**: 用户可以查询具体数据的参考依据
3. **反馈权**: 用户可以对数据的准确性提出反馈
4. **导出权**: 用户可以导出自己的训练和营养数据

### 平台责任

1. **准确性**: 确保数据的准确性和科学性
2. **及时性**: 定期更新和维护数据
3. **透明性**: 公开数据来源和处理方法
4. **安全性**: 保护用户数据的安全和隐私

### 免责声明

1. **个体差异**: 健身和营养建议因人而异，用户应根据自身情况调整
2. **专业咨询**: 有健康问题的用户应咨询专业医生或营养师
3. **数据误差**: 营养数据可能存在测量误差，仅供参考
4. **风险提示**: 健身运动存在受伤风险，用户应注意安全
